# Configuration Guide

Gition uses a flexible configuration system that allows you to customize your workspace to fit your project's needs.

## Configuration File

The main configuration file is located at `.gitionrc/config.yaml` in your project root. This file controls all aspects of your Gition workspace.

## Basic Configuration

```yaml
# Project Information
name: "My Project"
description: "Project documentation and task management"

# Directory Structure
docsDir: "docs" # Where documentation files are stored
tasksDir: "tasks" # Where task files are stored

# Default Language
defaultLanguage: "en" # en, fr, or es
```

## Theme Configuration

Customize the appearance of your workspace:

```yaml
theme:
  # Primary color (CSS color value)
  primaryColor: "#3b82f6"

  # Default theme mode
  darkMode: true

  # Custom CSS variables
  variables:
    "--border-radius": "8px"
    "--font-family": "Inter, sans-serif"
```

## Feature Flags

Enable or disable specific features:

```yaml
features:
  # Core Features
  taskManagement: true # Enable task board and management
  realTimeUpdates: true # Enable file watching and live updates
  search: true # Enable global search functionality

  # UI Features
  darkModeToggle: true # Show dark/light mode toggle
  languageSelector: true # Show language selection dropdown
  sidebarCollapse: true # Allow sidebar to be collapsed

  # Advanced Features
  codeHighlighting: true # Syntax highlighting for code blocks
  mermaidDiagrams: false # Mermaid diagram support (coming soon)
  mathRendering: false # LaTeX math rendering (coming soon)
```

## Task Types

Define custom task types with their own folders and icons:

```yaml
taskTypes:
  - name: "epic"
    folder: "epics"
    icon: "target"
    color: "#8b5cf6"

  - name: "story"
    folder: "stories"
    icon: "book"
    color: "#3b82f6"

  - name: "bug"
    folder: "bugs"
    icon: "bug"
    color: "#ef4444"

  - name: "doc"
    folder: "documentation"
    icon: "file-text"
    color: "#10b981"
```

### Available Icons

Common icon names you can use:

- `target`, `book`, `bug`, `file-text`
- `check-circle`, `clock`, `flag`, `star`
- `user`, `users`, `settings`, `folder`
- `plus`, `minus`, `edit`, `trash`

Define team members for task assignment:

```yaml
    email: "john@example.com"
    avatar: "https://example.com/avatar.jpg"

  - name: "Jane Smith"
    email: "jane@example.com"
    role: "Product Manager"

  - name: "Bob Wilson"
    email: "bob@example.com"
    role: "Developer"
```

## Environment Variables

You can also configure Gition using environment variables:

```bash
# Basic Configuration
GITION_NAME="My Project"
GITION_DOCS_DIR="documentation"
GITION_TASKS_DIR="project-tasks"
GITION_DEFAULT_LANGUAGE="fr"

# Theme
GITION_THEME_PRIMARY_COLOR="#10b981"
GITION_THEME_DARK_MODE="true"

# Features
GITION_FEATURE_TASK_MANAGEMENT="true"
GITION_FEATURE_REAL_TIME_UPDATES="true"
GITION_FEATURE_SEARCH="false"

# Development
GITION_DEBUG="true"
GITION_LOG_LEVEL="info"
```

Environment variables take precedence over config file settings.

## Advanced Configuration

### Custom Workspace Structure

For projects with complex structures:

```yaml
workspaces:
  - name: "Frontend"
    docsDir: "frontend/docs"
    tasksDir: "frontend/tasks"

  - name: "Backend"
    docsDir: "backend/docs"
    tasksDir: "backend/tasks"

  - name: "Mobile"
    docsDir: "mobile/docs"
    tasksDir: "mobile/tasks"
```

### Custom Task Statuses

Define custom task statuses beyond the default todo/in-progress/done:

```yaml
taskStatuses:
  - id: "backlog"
    name: "Backlog"
    icon: "inbox"
    color: "#6b7280"

  - id: "todo"
    name: "To Do"
    icon: "circle"
    color: "#6b7280"

  - id: "in_progress"
    name: "In Progress"
    icon: "clock"
    color: "#f59e0b"

  - id: "review"
    name: "In Review"
    icon: "eye"
    color: "#8b5cf6"

  - id: "done"
    name: "Done"
    icon: "check-circle"
    color: "#10b981"
```

### File Processing

Configure how files are processed:

```yaml
processing:
  # File extensions to include
  includeExtensions: [".md", ".mdx", ".txt"]

  # Directories to ignore
  ignoreDirs: ["node_modules", ".git", ".next", "dist"]

  # Files to ignore
  ignoreFiles: [".DS_Store", "Thumbs.db"]

  # Maximum file size (in MB)
  maxFileSize: 10
```

## Configuration Validation

Gition validates your configuration on startup. Common validation errors:

- **Invalid directory paths**: Ensure `docsDir` and `tasksDir` exist
- **Missing required fields**: `name` is required
- **Invalid colors**: Use valid CSS color values
- **Unknown icons**: Use supported icon names
- **Invalid language codes**: Use "en", "fr", or "es"

## Configuration API

You can also update configuration programmatically via the API:

```bash
# Get current configuration
curl http://localhost:3000/api/config

# Update configuration
curl -X POST http://localhost:3000/api/config \
  -H "Content-Type: application/json" \
  -d '{"theme": {"primaryColor": "#10b981"}}'
```

## Best Practices

1. **Start Simple**: Begin with basic configuration and add complexity as needed
2. **Use Environment Variables**: For deployment-specific settings
3. **Version Control**: Include `.gitionrc/` in your git repository
4. **Team Consistency**: Agree on task types and statuses across your team
5. **Regular Updates**: Review and update configuration as your project evolves

## Troubleshooting

### Configuration not loading

- Check YAML syntax with an online validator
- Ensure file permissions allow reading
- Look for error messages in the console

### Features not working

- Verify feature flags are enabled
- Check browser console for JavaScript errors
- Ensure all required directories exist

### Performance issues

- Reduce `maxFileSize` if processing large files
- Add more directories to `ignoreDirs`
- Disable real-time updates for very large projects
